Fencing Results Reader PHP / Data parsing / Analytics

A web tool I built for the French Fencing Federation to automatically read competition result files, extract the key stats, and present them in a clear, coach-friendly dashboard. This replaced manual Excel work after every event.

01. Context Problem to solve

After each fencing competition, the federation receives raw result files. These files contain all bouts, rankings, pools, eliminations, etc. Before my tool, staff had to:

  • Open the files manually and search for relevant info (who beat who, score spreads, consistency).
  • Copy/paste into Excel to build internal reports for coaches and managers.
  • Repeat this for every category, every competition, every weekend.

This process was slow, error-prone, and depended on whoever was on duty. The federation wanted something repeatable, fast, and usable by non-technical staff.

02. Objective What I delivered

I built a lightweight PHP web interface that:

  • Ingests competition result files (export formats provided by the federation).
  • Parses and extracts structured data: athletes, ranking, match outcomes, indicators.
  • Computes performance metrics (for coaches and staff).
  • Displays everything on a clean web dashboard: no Excel formulas needed anymore.

I first demoed it live to the federation’s manager, then iterated based on feedback (readability, naming, and which KPIs matter most to them).

03. How it works Technical view

The pipeline is intentionally simple so it can run on any basic PHP hosting:

  • Upload / Select a file: the staff either uploads a results file or points to an existing one.
  • Parsing layer (PHP): I read the raw data line by line, detect blocks (pools, DE brackets, final standings), and map them to arrays / objects.
  • Stat engine: I generate key indicators such as victory margins, loss patterns, rank progression, etc.
  • Render: final info is displayed as HTML tables and summaries, so staff can screenshot / export directly.

No external database was required in the first version: all processing is done in memory, which makes deployment easier for a federation that doesn’t have internal devops resources.

04. Tech stack Tools I used

  • PHP for file parsing and server-side logic.
  • HTML / CSS for the dashboard UI.
  • Basic data structures in PHP (arrays, associative arrays) instead of a database to keep it portable.
  • Iterative design with the federation: they told me which KPIs they actually look at on Monday morning.

05. Impact Why it matters

After the demo, the federation could:

  • Generate a competition summary in minutes, not an hour+ of Excel cleaning.
  • Share consistent reports across coaches and staff — same format every time.
  • Focus discussions on athlete performance instead of “where did you get that number?”

This project also trained me to talk directly with non-technical stakeholders (federation managers, coaches) and translate what they mean by “we need stats” into something that can actually be computed reliably.
